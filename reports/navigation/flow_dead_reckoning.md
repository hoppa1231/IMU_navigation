# Flow dead reckoning

This report is generated by `src/build_flow_dead_reckoning.py`.

Method: train a ridge mapping from module sensor windows to local GPS velocity on all module flights except the test flight, then integrate predicted velocity from the first GPS point of the test flight. The real GPS trajectory is used only as the plotted reference and error target for the test flight.

- lookback window: `300` ms
- sensor sample: `20` ms
- ridge alpha: `100`

## Outputs

- trajectory CSV: `derived/predictions/flow_dead_reckoning/flow_dr.csv`
- HTML overlay: `artifacts/generated/flow_dead_reckoning/index.html`

- `flow_imu_alt` adds accelerometer, gyro, lidar and barometer aggregates.
